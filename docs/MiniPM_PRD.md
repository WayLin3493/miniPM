# MiniPM PRD：美术 PM 每日学习助手

## 1. 产品概述

MiniPM 是一款面向游戏公司美术 PM 的每日学习与职业成长助手。产品通过每天短时学习、AI 总结、PMP 练习、工作场景化理解、收藏与复习，帮助美术 PM 在日常碎片时间里持续提升项目管理、游戏美术管线、AI 前沿和工具应用能力。

第一版产品形态为“网页后台 + 手机端网页/PWA”，面向个人自用，支持邮箱登录、云端数据存储和手机端安装到桌面。

核心价值：

- 学习内容 -> AI 总结 -> 工作应用 -> 收藏复盘 -> 职业成长闭环
- 每天约 10 分钟，降低学习压力
- 用轻松闯关式体验承载偏专业的学习内容

## 2. 目标用户与使用场景

目标用户：

- 游戏公司美术 PM
- 需要兼顾 PMP 项目管理知识、游戏美术专业知识、AI 前沿、工具链和开源项目
- 希望用低压力、短时间的方式保持学习习惯

主要使用场景：

- 每天上午收到学习提醒后，在手机端完成今日学习
- 利用碎片时间完成 5 道 PMP 练习题
- 阅读 4 条精选内容：美术知识、AI 前沿、GitHub 项目、AI 工具
- 完成全部任务后打卡，累计连续打卡天数
- 后台查看内容生成状态、抓取状态、AI 成本和学习数据

## 3. MVP 范围

第一版必须包含：

- 手机端 PWA
- 网页后台
- 邮箱登录
- 云端数据存储
- 每日自动发布内容
- PMP AI 练习题 5 道，作为一个刷题组
- 美术知识 1 条
- AI 前沿 1 条
- GitHub 项目 1 条
- AI 工具 1 条
- 阅读滑到底即完成
- 完成所有阅读和刷题后打卡
- 打卡天数与连续打卡统计
- 错题按遗忘曲线复习
- 每日三次提醒，完成打卡后停止当天后续提醒
- 后台查看内容、生成日志、抓取状态、AI 成本

第一版不包含：

- 行业案例拆解
- 原生 iOS/Android App
- 离线阅读
- 用户自定义标签
- 收藏备注
- 数据源配置后台
- 人工审核发布
- 个性化推荐算法

## 4. 每日内容结构

每天自动生成一个今日学习包：

| 类型 | 数量 | 说明 |
| --- | ---: | --- |
| PMP 练习题 | 5 道 | 单选 + 多选，AI 生成，标注“非官方 PMP 真题，仅作练习” |
| 游戏美术知识 | 1 条 | 公开网页内容 + AI 摘要 + 原文链接 |
| AI 前沿 | 1 条 | AI 新闻、论文、模型、工作流或工具趋势 |
| GitHub 项目 | 1 条 | 开源项目简介、应用价值、原始链接 |
| AI 工具 | 1 条 | 工具介绍、适用场景、预计上手成本 |

阅读类内容包含：

- 标题
- AI 摘要
- 完整公开正文或可提取正文展示
- MiniPM 工作应用提示
- 标签
- 预计阅读时间
- 原始链接

如果公开网页不允许抓取、正文不可提取或存在版权风险，则降级为“摘要 + 原文链接”。

## 5. 自动内容生成工作流

系统每天在上午 9 点前完成内容生成：

1. 定时任务启动。
2. 生成 5 道 PMP 练习题。
3. 从默认公开来源抓取美术知识、AI 前沿、GitHub 项目和 AI 工具候选内容。
4. 对候选内容进行去重、规则过滤和相关性筛选。
5. 使用低成本 AI 模型生成摘要、工作应用提示和标签。
6. 写入今日学习包。
7. 上午 9 点触发第一次提醒。
8. 后续提醒时间检查是否已打卡；已打卡则不再提醒。

内容源第一版由系统内置，不提供后台配置：

- GitHub 通过公开 API 或公开搜索结果获取项目数据。
- 美术知识、AI 前沿、AI 工具只抓取公开网页、公开 RSS 或公开文章。
- 不抓取公众号、付费内容、登录后内容，也不绕过网站限制。

## 6. 手机端体验设计

视觉方向：

- 多邻国式轻松学习体验
- “轻松学习小岛”概念
- 高绿色能量感
- 天空蓝、薄荷绿、柠檬黄、珊瑚粉作为辅助色
- 无大角色形象
- 通过小图标、路径、徽章、彩色卡片和奖励反馈营造趣味
- 避免工作台、SaaS 后台和考试压力感

核心页面：

| 页面 | 功能 |
| --- | --- |
| 今日小冒险 | 今日学习路径、完成进度、连续打卡、继续学习 |
| PMP 挑战 | 5 道题、单选/多选、即时判题、PM 小贴士 |
| 阅读详情 | 课程封面、Mini 总结、正文、原文链接、滑到底完成 |
| 今日打卡 | 完成反馈、成长能量 +1、连续打卡 |
| 复习 | 到期错题复习 |
| 统计 | 打卡天数、连续打卡 |

完成规则：

- 阅读内容：滑到底即完成。
- PMP 刷题：5 道题全部作答即完成。
- 今日打卡：全部阅读内容完成且 PMP 刷题组完成。

## 7. 后台管理

后台用于个人运营和质量观察。

后台页面：

- 后台总览
- 今日内容
- 自动发布状态
- 抓取日志
- AI 成本
- 用户学习状态
- 错题和复习数据

后台第一版不提供内容源配置，但需要展示内置来源、抓取结果、失败原因和 AI token 成本。

## 8. 提醒与复习

每日提醒：

- 默认提醒时间：09:00 / 14:00 / 21:00
- 用户可修改提醒时间
- 如果当天已完成打卡，则停止当天后续提醒
- 第一版优先实现站内提醒 + 邮件提醒

错题复习：

- 答错题目自动进入错题库
- 使用固定遗忘曲线间隔：第 1 天、第 3 天、第 7 天、第 14 天
- 复习答对进入下一阶段
- 复习答错重置到第 1 阶段

## 9. 技术方案

推荐技术栈：

- 前端与后端：Next.js + TypeScript
- 样式：Tailwind CSS
- PWA：Web App Manifest + Service Worker
- 登录与数据库：Supabase Auth + Supabase Postgres
- 部署：Vercel Free
- 定时任务：GitHub Actions
- AI：优先使用低成本模型，控制在低 token 消耗模式
- 推送：第一版站内提醒 + 邮件提醒

预算目标：

- MVP 目标控制在 20 元人民币/月以内
- 托管平台优先使用免费档
- AI 生成采用候选过滤、短输入、结构化输出和缓存，避免全文批量送入模型

## 10. 成长路线

第一版采用固定四条成长路线，不做个性化算法：

1. PMP 基础线：项目管理判断力、风险识别、沟通与进度控制。
2. 美术管线线：角色、场景、动画、TA、外包、验收、资产规范。
3. AI 工具线：图像生成、视频生成、参考整理、自动化工作流。
4. GitHub 工具线：开源工具发现、工作流优化、技术辅助理解。

每条内容写入一个成长路线标签，未来可用于统计和会员功能。

## 11. 商业化预留

第一版面向个人自用，但保留未来会员扩展空间：

- 多用户账号结构
- 会员字段预留
- 内容生成成本统计
- 管理后台权限区分
- 可扩展数据表结构

未来会员可考虑：

- 更多每日内容
- 私人知识手册
- 个性化成长路线
- 高级复习计划
- 更多 AI 工具工作流

## 12. 验收标准

MVP 完成时应满足：

- 用户可以通过邮箱登录。
- 手机端可以安装为 PWA。
- 每天可以看到 5 道 PMP 题和 4 条阅读内容。
- PMP 题可以判题并进入错题复习。
- 阅读滑到底可记录完成。
- 完成所有任务后可以打卡。
- 统计页显示打卡天数和连续打卡。
- 后台可以看到今日内容、日志、抓取状态和 AI 成本。
- 定时任务可以自动生成每日内容。
- 默认提醒时间为 09:00 / 14:00 / 21:00。
- AI 题目明确标注“非官方 PMP 真题，仅作练习”。
